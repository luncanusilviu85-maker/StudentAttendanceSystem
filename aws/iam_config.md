# IAM Configuration for Student Attendance System

## EC2 Instance Role Policy

Attach the following inline policy to the IAM role assigned to the EC2 instance.
The application uses **instance profile credentials** — no access keys are stored in code or environment variables.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "S3ReportAccess",
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:ListBucket",
        "s3:DeleteObject"
      ],
      "Resource": [
        "arn:aws:s3:::attendance-reports-bucket",
        "arn:aws:s3:::attendance-reports-bucket/reports/*"
      ]
    },
    {
      "Sid": "CloudWatchLogs",
      "Effect": "Allow",
      "Action": [
        "logs:CreateLogGroup",
        "logs:CreateLogStream",
        "logs:PutLogEvents",
        "logs:DescribeLogStreams"
      ],
      "Resource": "arn:aws:logs:eu-west-1:*:log-group:/attendance-system/*"
    },
    {
      "Sid": "RDSConnect",
      "Effect": "Allow",
      "Action": [
        "rds-db:connect"
      ],
      "Resource": "arn:aws:rds-db:eu-west-1:*:dbuser:*/admin"
    }
  ]
}
```

## S3 Bucket Policy (Block Public Access)

The S3 bucket must have **Block All Public Access** enabled.
Files are served only via pre-signed URLs (7-day expiry) generated by the application.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyPublicAccess",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::attendance-reports-bucket/*",
      "Condition": {
        "StringNotEquals": {
          "aws:PrincipalType": "AssumedRole"
        }
      }
    }
  ]
}
```

## RDS Security Group

- Allow inbound **TCP 3306** from the EC2 security group only.
- No public endpoint (set `Publicly Accessible = false`).
- Enable **storage encryption** (AES-256) and **automated backups** (7-day retention).

## Notes

- Rotate credentials using IAM roles; never use long-lived access keys.
- Enable **MFA** on all IAM user accounts.
- Use **AWS Secrets Manager** for the DB password instead of environment variables in production.
